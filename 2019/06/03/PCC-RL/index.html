PCC-RL

Posted on 2019-06-03 | In paper | Visitors: | Readers 人

  • here for paper

  • Author: Nathan Jay

  • Congegstion control, Reinforcement learning

摘要 (abstract)：

用RL的做法改进了PCC-Vivace, 并且详细讨论了使用RL来tackle cc problem 会带来的fairness,safety and generalization 问题.

动机（motivation):

他首先列举了一个sing-bottleneck的dumbell拓扑结构.并说明了Endhost端调整sending-rate的困难性。Endhost端调整sending-rate的时候最大的问题是decentralized & uncoordinated所以每个endhost 都不知道有多少个sender端在compete，有多少的bottleneck bandwidth。

使用RL来tackle congestion control问题,我们假设过去的history信息中包含了网络network的一些信息，在这些信息里我们可以学习到针对不同的网络,做怎样的mapping

考虑能学到的两种pattern【这个我也能造出来】

  1. 辨别非congestion导致的loss 和congestion 导致的loss （和TCP相比）
  2. 在上面这种情况中，他设置了一个随机丢包的网络，CUBIC和PCC-RL相比，不断的砍窗口，因此效果极差
  3. 适应多种不同的网络状况 (和TCP相比)
  4. 在上面这种情况中，他设置了一个bottleneck不断变化的网络情况，TCP和PCC-RL相比，可以看到档网络变化的时候，完全打不满带宽。

PCC-RL also called Aurora

实现（impletmentaion):

RL 四关键元素的定义:

  1. Actions: changes to sending rate

  2. States: bounded history of network statistics.

    • latency gradient
    • latency ratio
    • sending ratio

    选择上述向量的依据是防止高度变化的元素（比如latency的值来影响模型的泛化能力）

  3. State 其实是一个fixed-length history的vector.
    St=(Vt-k+d,...,Vt-d)

    这个d代表的是一个差值，这个差值d是收集到反馈信息和发出信息的时间差

  4. Setting Rewards: depends on the specific application. Specific reward functions. (3.1)

  5. discount factor r: this value should be in Section 3.3 (不能太小至少0.5)
    r=0.99的时候收敛更快

Aurora 介绍:

在这个部分，我们介绍了Aurora: RL of CC

Architecture

  1. RL inputs and outputs: 2选1 （有一个小因子的防震荡因子a）
  2. Neural network: 神经网络 (一个简单的结构已经可以得到很好的结果)
  3. Reward function: 选择了简单的linear function of throughput and latency and loss.

Training:

在一个open-source gym 的环境里做了训练,使用的PPO算法

Choice of Parameters

  1. k (在state向量里考虑几轮的vector结果值) k=2 与 k=10会比较接近 选择2
  2. r (discount factor)

Evaluation:

我们训练的结构使用简单的单一网络路径，有不同的参数。我们的测试结合包含了各种不同的链路情况。and
training in an emulated rather than simulated environment. 特别是，我们使用标准网络研究工具(Mininet(Fontes等，2015)和Pantheon(Yan等，2018))测试我们训练模型的性能和稳健性，用于各种网络场景，并提供比较最先进的拥塞控制协议.

bandwidth, latency, queue size, random loss rate, 这四个参量可以模拟所有的网络情形。

  1. bandwidth sensitivity 1.2 Mbps - 6 Mbps
  2. Latency sensitivity 50ms - 500ms
  3. Queue sensitivity 2 - 2981 packets
  4. loss sensitivity 0% - 5% loss
  5. dynamic link

Evaluation result:

  1. First, Aurora is surprisingly robust to environments outside the scope of its training.
  2. Second, Aurora is comparable to or outperforms the state-of-the-art in congestion control (BBR, PCC-Vivace, RemyCC, and Copa).

5. A Testbed for RL Congestion Control

  1. 5.1 Links (simple FIFO Queues)
  2. 5.2 Packets (sender latency is_dropped)

6. Remaining Challenges:

fairness (with tcp 有很差的效果)

TCP friendliness 做的不令人满意

Multiple Objectives:

怎么去实现一个special的Reward Function. 能满足各种用户的不同指标。
拥塞控制中的目标函数还是一个争议性的话题。最小化延迟，最大化带宽和其他的表现。
多目标的RL是一个结构来处理不同价值因素之间的tradeoff的好方法。但还不知如何实现。

Generalization and Adaptation to changes in the network

虽然结果还是令人满意.但是针对网络不断变化的适应性的问题始终是一个有趣的问题。

the problem is called generalization 泛化
